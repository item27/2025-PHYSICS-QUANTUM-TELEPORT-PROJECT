# Проект: Quantum Teleportation Visualizer (MVP)

## 1. Назначение документа
Кратко описывает идею, цели и рамки проекта Quantum Teleportation Visualizer для разработчиков и участников команды.

Документ задаёт смысл и границы разработки и не является технической спецификацией.

## 2. Общая идея
Веб-приложение, которое наглядно объясняет протокол квантовой телепортации одного кубита. Проект не моделирует реальную физику и не претендует на точные симуляции — его задача показать идею.

## 3. Аудитория
- Люди без физического образования: простой язык, подсказки и пошаговая визуализация.
- Люди, знакомые с темой: корректные термины, уважение к физике и каноническая последовательность шагов.

## 4. Что такое квантовая телепортация в контексте проекта
Телепортация — перенос квантового состояния с помощью заранее подготовленной запутанной пары, измерения, передачи классической информации и восстановления состояния. Ключевые акценты: переносится состояние, исходное состояние уничтожается, скорости света не нарушаются, копирования нет.

## 5. Ключевая идея
«Квантовая телепортация — это информационный протокол, который можно понять визуально». Приложение показывает причинно-следственные связи и объясняет, зачем нужен каждый шаг.

## 6. Режимы работы
- **Демонстрационный**: автономный, без backend, упрощённая модель состояния, акцент на наглядности.
- **Сетевой**: с backend и WebSocket, каждый клиент представляет свою роль, состояния синхронизируются в реальном времени.

## 7. Ограничения MVP
- Телепортируется только один кубит.
- Нет квантовых библиотек, шумов и декогеренции.
- Важнее ясность UX, чем физическая точность.

## 8. Что проект не делает
Не телепортирует объекты, не моделирует квантовый компьютер, не решает уравнения и не заменяет учебник. Это визуальный объяснитель.

## 9. Ожидаемый эффект
После использования пользователь понимает, что такое квантовая телепортация, почему она не нарушает физику и чем отличается от фантастики, и может кратко объяснить процесс другому человеку.

## 10. Философия UX
Интерфейс дружелюбный, без лишних терминов, поощряет исследование и выглядит современно. Принципы: «Сначала показать, потом объяснить», «Меньше текста — больше смысла», «Без магии без пояснений».

### 10.1 Эталон шагов протокола
Единый список шагов, на который опираются тексты, UI и API:

1. Подготовка состояния $|\psi\rangle = \alpha|0\rangle + \beta|1\rangle$ — локальная суперпозиция.
2. Пара Белла $|\Phi^+\rangle = (|00\rangle + |11\rangle)/\sqrt{2}$ — запутанность через $H$ и CNOT.
3. Распределение: $|\psi\rangle_A \otimes |\Phi^+\rangle_{CB}$ — запутанность сохраняется между B и C на расстоянии.
4. Беллово измерение: CNOT + H — подготовка к измерению, чтобы двух битов хватило для коррекции.
5. Коллапс: вероятности $P_{ij} = |c_{ij}|^2$, появляются классические биты $b_1, b_2$.
6. Классический канал: оператор $X^{b_2} Z^{b_1}$ определяется по результатам измерения.
7. Коррекция у Боба: применения $X$ и $Z$ восстанавливают состояние.
8. Итог: $|\psi\rangle$ восстановлено у Боба, исходное состояние уничтожено.
